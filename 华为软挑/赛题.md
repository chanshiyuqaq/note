# 得到NFP之后的算法

## 二、算法实现步骤详解

### 2.1 第一步：确定参考点的位置关系

在获得NFP之后，首先需要确定多边形B的参考点与NFP的位置关系。这个步骤包含两个子任务。首先是参考点的确定，在NFP生成过程中，通常选择多边形B顶点列表中的第一个顶点作为参考点，并追踪该点在B绕A运动过程中的轨迹来形成NFP。因此，在计算最小平移向量时，必须使用与NFP生成时完全相同的参考点定义。其次是位置判断，利用射线交叉算法（Ray Crossing Algorithm）判断参考点相对于NFP的位置：点在NFP外部返回"外部"，点在NFP边界上返回"边界"，点在NFP内部返回"内部"。

如果判断结果显示参考点位于NFP外部，则直接输出零向量（0.00000, 0.00000），表示两多边形原本就不重叠，无需任何平移。如果参考点恰好在NFP边界上，同样输出零向量，因为此时两多边形虽然接触但并未重叠。只有当参考点位于NFP内部时，才需要进入后续步骤计算最小平移向量。

### 2.2 第二步：寻找最近边界点

当参考点位于NFP内部时，需要找到从该点到NFP边界的最短距离。这个问题可以转化为一个优化问题：在NFP边界上所有点中，找到与参考点欧氏距离最小的点。

具体的实现思路是遍历NFP的所有边。对于NFP的每一条边，计算从参考点到该边的垂直距离。这里需要考虑两种情况：如果参考点到边的投影点落在线段内部，则使用点到线段的距离公式计算垂直距离；如果投影点落在线段外部，则最近距离为参考点到线段两个端点中较近的那个端点的距离。在所有边中找到距离最小的边和对应点，该点即为NFP边界上距离参考点最近的点。

### 2.3 第三步：计算平移向量并确定方向

一旦找到NFP边界上的最近点，就可以计算平移向量。最直接的向量就是从参考点指向最近点的向量，即向量 = 最近点 - 参考点。这个向量的长度就是最短分离距离，其方向就是分离的最优方向。

然而，需要特别注意的是平移方向的正确性。在NFP的定义中，我们假设多边形A保持固定，多边形B绕A运动生成NFP_AB。那么当我们需要分离两个重叠的多边形时，平移向量应该将B从当前位置移动到使其参考点落在NFP边界上的位置。因此，平移向量的方向应该是从参考点指向NFP边界，即朝外指向NFP。

### 2.4 特殊情况处理

在实际计算中，存在一些需要特别处理的边界情况。首先是参考点恰好在NFP顶点上的情况，此时最近边界点就是该顶点本身，平移向量为零向量。其次是多个边具有相同最小距离的情况，此时任意选择一个都是正确的，因为它们的长度相同。最后是多边形B与A完全包含的情况，此时NFP内部可能包含多个区域，需要确保算法能够正确处理。

## 三、算法复杂度分析

### 3.1 时间复杂度

从算法步骤可以看出，整个求解过程的时间复杂度主要取决于NFP的顶点数和计算最近边界点时对边的遍历。假设NFP具有m个顶点，判断点是否在多边形内的时间复杂度为O(m)，寻找最近边界点需要遍历所有边，时间复杂度也为O(m)。因此，对于每个测试样本点，算法的总时间复杂度为O(m)。

### 3.2 预处理的优势

赛题任务书中提到允许1秒的预处理时间，这为算法优化提供了重要机会。在预处理阶段，可以预先计算和存储一些几何信息。例如，可以预先构建NFP的边段数据结构，存储每条边的端点坐标和法向量，以便在后续计算中快速查询。此外，还可以预先计算NFP的边界框（Bounding Box），用于快速排除明显不需要考虑的边。
