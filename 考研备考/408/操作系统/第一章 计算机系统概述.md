
# 基本概念

操作系统负责**管理硬件资源为应用程序提供运行环境** , 充当硬件与用户之间的桥梁

**四大核心功能**：文件管理，设备管理，存储器管理（内存），处理器管理（进程/线程）

两个视角：
操作系统作为**计算机系统资源的管理者**

操作系统作为**用户**与计算机硬件系统之间的接口：
有三类用户接口：
- 联机用户接口（命令行接口）
- 脱机用户接口（作业控制语言）用户将作业控制命令写入作业说明书，连通作业一并提交；系统调度到该作业的时候，自动解释并执行说明书中的命令，无需用户实时干预。
- 图形用户接口

程序接口：
	核心是一组**系统调用**，早期操作系统调用基于汇编语言实现，仅能在汇编程序中直接使用。
	在高级语言中(C语言)通过标准库函数**间接**触发系统调用
	标准库函数是对系统调用的封装

操作系统的特征：
操作系统的基本特征包括：共享，虚拟，并发，异步。
- 并发与并行：
	并发指的是多个事件在**同一时间间隔**内发生。在多道程序环境下，操作系统**通过调度机制**实现多道程序的交替执行，保持CPU高利用率
	并行：指的是多个事件在同一时刻真正发生，需要多核CPU，多处理机等硬件支持 。
- 共享：指的是系统中的资源可供并发进程共同使用
	两种共享方式：
	互斥共享，保证在任意时间段内仅有一个进程访问该资源
	（打印机）
	同时访问：指的是宏观上同时访问，微观上采用分时交替方式，即分时共享
- 虚拟：
	虚拟指的是通过某种技术手段将一个物理上的实体映射为多个逻辑上的对应物。
	多道程序设计技术->每个程序仿佛独占CPU
	虚拟存储器技术->每个程序仿佛独占内存
	虚拟设备技术->每个用户独占一台逻辑设备
	原本需要互斥访问的资源可转化为可并发使用的共享资源
- 异步(Asynchronism)：
	进程的执行以不可预知的速度断续推进
	需使用操作系统提供的同步机制

# 操作系统的运行环境

什么是操作系统内核：
将与硬件紧密相关的模块（中断处理程序，设备驱动程序）
运行频率较高的模块（时钟管理，进程调度）
以及各个模块共用的基础操作集中置于内存中，

操作系统内核的两类功能：
- 支撑功能
	时钟管理
	中断机制
	原语
	- 原语是一个操作，由若干条指令构成。具有原子性，常通过关中断等手段确保
- 系统资源管理功能
	进程管理
	设备管理
	内存管理（存储器管理）

	严格区分操作系统代码与用户程序的运行环境，绝大多数计算机系统通过硬件支持实现区分
0表示内核态，1表示用户态。
	通过区分用户态与内核态，可对指令分类进行保护，有一些指令只能在内核态执行，称为**特权指令**（可访问全部地址空间），其余称为非特权指令，可在任意模式下执行。


中断（外中断）与异常（内中断）
中断指的是来自CPU执行指令**外部的事件**，IO中断，时钟中断
异常指的是来自CPU执行指令内部的事件，地址越界，运算溢出，缺页异常

查看课本：中断与异常的分类，硬件中断与软件中断

中断和异常的处理流程：

对于中断，CPU执行完某条指令，检查中断，发现中断请求信号，关中断，保留现场，跳转到中断处理服务程序，恢复现场，开中断
对于异常，CPU在执行某个指令中发生，进行处理后重新执行本条指令。

**中断处理程序的入口地址由中断向量表提供**
- 中断向量表在内存中，且硬件能直接定位
- 拿到向量表基地址后，CPU使用中断号做索引，直接计算出对应条目的内存地址，读出中断处理程序的入口地址
- 将其加载到PC寄存器，下一条就会自动执行
- 通过中断返回指令触发硬件恢复现场 

**访管中断**
访管中断是用户程序通过访管指令**主动请求操作系统服务**，从用户态切换到核心态执行系统调用的自愿中断。
```cpp

#include <unistd.h>

int main() {
    char buffer[100];
    // read是典型的访管命令
    // 用户程序通过此命令请求内核读取文件
    int bytes_read = read(0, buffer, sizeof(buffer)); // 从标准输入读取
    return 0;
}

```

**操作系统中的通道技术（DMA）**

又称为I/O处理机或I/O控制器，独立于CPU运行，能够自主管理外部设备与内存之间的数据传输，从而实现CPU与I/O操作的并行工作，极大地提高了系统的整体效率。
通道作为一个独立的处理单元，具有自己的指令系统、程序计数器和状态寄存器。它通常由三部分组成：通道控制器、通道程序存储器和数据缓冲寄存器。当系统需要进行I/O操作时，**CPU只负责启动通道和等待操作完成**，而具体的传输过程则由通道独立执行。

**PSW是什么**
程序状态字（包含特权级标志、条件码等）


**对比一般过程调用与系统调用**
系统调用的被调用过程在内核态，
一般过程调用与调用程序**运行在同一个状态**，可能是系统态，也可能是用户态
中断需要压栈PC,PSW 
一般过程调用压栈PC

**当发生中断时候由硬件完成的有**：
- PC,PSW
- CPU的模式的更改

时钟中断服务程序工作任务详解
- 每个进程都被分配了一个固定长度的时间片（通常为几十毫秒）。时钟中断程序负责递减当前进程的时间片计数器，当计数器归零时，触发进程调度。
- 负责维护精确的系统时间
- 进程调度触发
- 当前进程占用CPU的时间(2018统考)


概念详解：
- 中断隐指令
	中断发生时候硬件自动执行的，
	包括保存PC，PSW，切换CPU特权级
	将PC，PSW保存在内核栈中，**同时恢复也是由硬件完成的**
- 屏蔽中断指令(关中断指令)
	在进程切换时候
- 修改程序状态字(PSW)指令
	在内核态下修改
- 中断屏蔽字设置指令->必须在内核态执行
- 保存中断屏蔽字


# 操作系统结构

宏内核：将所有的功能模块集成在一个紧密耦合的整体中，运行在内核态，模块共享内存，减少上下文切换，消息传递开销，有性能优势。缺点是不便于维护。
微内核：将核心内容保留在内核，非核心功能移动至用户态运行。
外核：将物理资源暴露给用户级系统，自身仅负责资源的安全分配与隔离。

- 微内核添加新的系统服务，无需修改内核代码
- 分层式设计便于调试
- 微内核中有什么？
	- 中断相关
	- 进程调度
	- 存储器管理，（逻辑地址到物理地址的映射
	- 低级IO
- 加载系统的时候BIOS在磁盘上，引导阶段建立中断向量表，内核初始化阶段完成进程调度相关操作。MBR(主引导记录) 负责确定活动分区，加载活动分区引导扇区(分区引导记录)
- 虚拟机可以通过软件实现也可以通过硬件实现
