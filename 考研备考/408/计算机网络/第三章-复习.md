127.0.0.1
在 IPv4 分类中，A 类网络号 `127`（即 `127.0.0.0/8` 这个巨大的网段，包含 1600 多万个地址）被全部保留，定义为“环回网段”（Loopback）。


#### 它是如何工作的？

当你在电脑上写了一个本地测试程序，让“浏览器”访问本地的“Web 服务器”：

- **目的 IP**：你输入 `127.0.0.1`（目的地址）。
- **源 IP**：操作系统会自动把源 IP 也设为 `127.0.0.1`。

#### 数据流向（闭环）：

1. 浏览器发送数据包（源：`127.0.0.1`，目的：`127.0.0.1`）。
2. 操作系统网络栈看到目的 IP 是 `127.0.0.1`，**立刻意识到这是给自己的**。
3. 操作系统**直接在内存中把数据包转弯**，送回给本地的 Web 服务器。
4. **这个数据包全程根本没有通过网线发出去，甚至都没有到达你电脑的物理网卡芯片**。它在操作系统内部就完成了“自发自收”。

OSPF
链路状态路由算法，需要知道整个链路的状态

- 每个路由器只描述“我和谁相连、代价是多少”
- 这些描述被洪泛给全区域所有路由器
- 所有路由器拼出同一张拓扑图
- 每台路由器再以自己为根计算最短路径

相关路由器会立即产生新的 LSA，并在区域内洪泛。
所有路由器收到新 LSA 后更新 LSDB，然后重新运行 SPF。
所以 OSPF 对变化响应更快。

RIP属于应用层 使用UDP 端口 520
- 跳数越少越好
- 合法路径距离不能超过15
- 一般用于小型自治系统
定时 和邻居 交换路由表

新路只有更近才采纳，老路如果距离有变化，则以最近的信息为准
这里的新路和老路指的是 针对某一目标网络 的下一条路由器是否有变化，A还是A则是老路，A换成B则表示新路

当网络的拓扑结构变化的时候，按照新的拓扑结构 进行消息的互传

OSPF
传播信息使用洪泛法，邻居传邻居
不可以回流，且**不会二次传递**
一旦探测到变化就立即再次洪泛
允许自治系统设计规则计算 **代价**
默认使用带宽，等于： 默认带宽/线路实际带宽 = 链路实际带宽越小，计算出的代价越大
OSPF协议规定：
路由器->直连网络的代价为1
直连网络到路由器的代价为0
同样，支持**多条等价线路**可用于负载均衡，从不同线路转发
有路由鉴别功能
在洪泛信息的时候标注版本号（32b）根据序号，可以分清那个是最新的状态

划分区域:为 **主干区域** 和 **非主干区域**
非主干区域通过路由器连接到主干区域上

这种情况下，路由器分为三类，分别是

**自治系统边界**路由器 负责 自治系统之间的IP分组 路由转发
**区域边界**路由器 负责**流出流入该区域**的分组转发
**区域内部**路由器 区域内部路由器，负责内部分组转发

区分LSDB(整个邻接链表)
    LSA（邻接链表的某一行）
    LSI（泛指 链路状态）
