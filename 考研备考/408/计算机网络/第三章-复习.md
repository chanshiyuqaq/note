127.0.0.1
在 IPv4 分类中，A 类网络号 `127`（即 `127.0.0.0/8` 这个巨大的网段，包含 1600 多万个地址）被全部保留，定义为“环回网段”（Loopback）。


#### 它是如何工作的？

当你在电脑上写了一个本地测试程序，让“浏览器”访问本地的“Web 服务器”：

- **目的 IP**：你输入 `127.0.0.1`（目的地址）。
- **源 IP**：操作系统会自动把源 IP 也设为 `127.0.0.1`。

#### 数据流向（闭环）：

1. 浏览器发送数据包（源：`127.0.0.1`，目的：`127.0.0.1`）。
2. 操作系统网络栈看到目的 IP 是 `127.0.0.1`，**立刻意识到这是给自己的**。
3. 操作系统**直接在内存中把数据包转弯**，送回给本地的 Web 服务器。
4. **这个数据包全程根本没有通过网线发出去，甚至都没有到达你电脑的物理网卡芯片**。它在操作系统内部就完成了“自发自收”。

OSPF
链路状态路由算法，需要知道整个链路的状态

- 每个路由器只描述“我和谁相连、代价是多少”
- 这些描述被洪泛给全区域所有路由器
- 所有路由器拼出同一张拓扑图
- 每台路由器再以自己为根计算最短路径

相关路由器会立即产生新的 LSA，并在区域内洪泛。
所有路由器收到新 LSA 后更新 LSDB，然后重新运行 SPF。
所以 OSPF 对变化响应更快。

OSPF
传播信息使用洪泛法，邻居传邻居
不可以回流，且**不会二次传递**
一旦探测到变化就立即再次洪泛（每个节点针对一个消息只洪泛一次）
允许自治系统设计规则计算 **代价**
**默认使用带宽**，等于： 默认带宽/线路实际带宽 = 链路实际带宽越小，计算出的代价越大
OSPF协议规定：
路由器->直连网络的代价为1
直连网络到路由器的代价为0
同样，支持**多条等价线路**可用于负载均衡，从不同线路转发
有路由鉴别功能

假设使用MAC路由
- 如果用 MAC 路由，全球的路由器必须记录**每一个** MAC 地址应该往哪转。目前全球有数百亿台设备，路由器的内存会瞬间被撑爆（**路由表暴涨**），查表速度会慢如蜗牛。
- 如果网络层用 MAC 寻址，整个互联网的路由器都要紧急更新：“这个 MAC 已经去上海了，大家以后把发给它的包往上海转。”这会造成全球路由震荡。
- 并非所有物理链路都使用**以太网的 48 位 MAC 地址**。比如广域网常用的 PPP 协议、帧中继、ATM 协议，它们的**硬件寻址方式完全不同**，甚至根本没有 MAC 地址。
- 如果全球是一个巨大的 MAC 网络，一个广播包（如“谁是张三？”）将会被复制并发送给全球的每一台设备。
