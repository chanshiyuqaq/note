## 深度优先搜索

``` cpp

int V;      //顶点数
vector<vector<int>>store; //存储邻接关系
vector<bool>is_visited;

对于一个联通分支的搜索：

void dfs(int s)
{
	is_visited[s] = 1;
	for(int i = 0;i<store[s].size();i++)
	{
		if(!is_visited[store[s][i]])
		{
			bfs(store[s][i]);
		}
	}
}
```

判断题：
- 对于非强连通图，必须调用两次及以上的BFS才能访问所有顶点？ （错）
	举三个点的特例
- 使用DFS和拓扑排序可以判断有向图中有没有环 （对）
	DFS可以使用返回边判断
	（遍历过程中遇到了“正在访问中”（处于递归栈中）的节点）
	拓扑排序呢？
	拓扑排序成功（输出序列长度 = 总节点数）   ⟺  ⟺ **无环**。
	拓扑排序失败（输出序列长度 < 总节点数）   ⟺  ⟺ **有环**。
- 如果图中任意一个点的入度都不为零，则图中一定有环吗（对）
	假设图中有 nn 个节点。我们从任意一个节点 v1v1​ 开始，逆着箭头的方向（沿入边）向上寻找：

	- 因为 v1v1​ 的入度不为零，所以一定能找到一个指向它的点 v2v2​。
	- 因为 v2v2​ 的入度也不为零，所以一定能找到一个指向它的点 v3v3​。
	- 以此类推，我们可以得到一个无穷的序列 ⋯→vk→⋯→v2→v1⋯→vk​→⋯→v2​→v1​。

	但是，**图的节点数是有限的**（nn 个）。当你找第 n+1n+1 个点时，根据**鸽巢原理**（抽屉原理），这 n+1n+1 个点中一定至少有一个节点是重复出现的。

	- 一旦某个节点重复出现（例如 vi=vjvi​=vj​），就意味着你在追踪过程中回到了原点，从而形成了一个环。
