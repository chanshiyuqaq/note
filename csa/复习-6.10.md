### 第八章 多处理机

多处理机系统由多个独立处理机组成，每个处理机都能独立执行自己得程序
分类: 
- 紧密耦合|松散耦合
- 共享存储器|分布存储器
- 处理机类型：同构型(多个处理器结构基本相同) | 异构型
- 处理机个数：
	- 大规模并行处理机MPP （大量节点组成，每个节点有自己的处理器和本地内存）
	- 对称多处理机SMP(多个处理机共享内存，地位对等)
- PE与IOP之间互联方式：
	- **PE**->Processing Element 处理单元
	- **IOP**->I/O Processor 输入输出处理机
	- 对称 | 
	- 非对称(只有某些处理单元和输入输出处理机相连) | 
	- 冗余对称(在非对称的基础上增加冗余的IOP)
- 存储器访问方式：
	- 均匀（Uniform Memory Access）访问内存一样快
	- 非均匀（Non Uniform Memory Access）访问内存内地快远程慢
	- 只有cache(Cache Only Memory Access) 本地内存当cache

属于**多指令流，多数据流**系统
多个处理机之间有机间互联网络实现通信，实现任务，作业之间的并行

| 对比维度      | 并行处理机                                         | 多处理机                                             |
| --------- | --------------------------------------------- | ------------------------------------------------ |
| 结构灵活性     | 针对向量、数组处理而设计的，有专用性，虽然处理单元数很多，但设置有限的、固定的机间互连通路 | 实现作业、任务、程序段的并行，适应算法，结构灵活多变，实现复杂的机间互连，避免争用共享的硬件资源 |
| 程序并行性     | 实现操作级并行，**并行性存在指令内部**                         | **并行性还存在于指令外部**，表现于多个任务间的并行                      |
| 并行任务派生    | 通过指令来反映数据间是否并行计算，并由指令直接启动多个处理单元并行工作           | 需要**专门的指令或语句指明程序中各程序段的并发关系**，并控制并发执行             |
| 进程同步      | 实现指令内部对数据操作的并行                                | 实现指令、任务作业级并行                                     |
| 资源分配和任务调度 | 处理单元数目固定，利用屏蔽手段，改变数目                          | 处理机数目不固定，复杂                                      |

### 互连网络

互联网络是并行处理系统的核心组成部分
由**开关元件**按照**一定**的**拓扑结构和控制方式**构成的网络



图中每台处理机与 **本地存储器，私有高速缓存** 相连
![[Pasted image 20260610225907.png|413]]
考虑该图中的网络有哪些？
- IPMN->处理机存储器网络
- IPCN->处理机之间的通信网络
- PION->处理机和IO之间的网络

在拓扑上，互连网络是 输入节点到输出节点之间的一组互联或者映射
是**SIMD MIMD**的关键组成部分

三大要素：
互连结构
开关元件
控制方式

互联网络有N个输入端N个输出端
n = logN
使用n位二进制位来表示N个**输入端和输出端的二进制地址**

恒等函数：相同编号的输入端和输出端**一一对应互连**所实现的置换
交换函数：实现二进制地址编码中第K位互反的**输入端与输出端之间的连接**
