## 一、修改策略概述

针对第一份报告中识别的 **5个 Blocker、3个 Critical、9个 Major** 级别问题，本次修改遵循以下原则：
1. **资源安全**：所有 Cursor、Stream 均采用 `try-finally` 或 `try-with-resources` 确保关闭
2. **消除重复**：提取 `existsInDatabase()` 公共方法，合并重复逻辑
3. **命名规范**：修正所有拼写错误（UNMOUONTED→UNMOUNTED, Foler→Folder, Tex→Text）
4. **防御式编程**：工具类添加 `private` 构造器；方法返回空集合替代 `null`
5. **简化复杂度**：将 `exportToText()` 拆分为 `exportFolders()` 和 `exportRootNotes()` 子方法

## 三、修改后质量评估

### 3.1 Blocker 级问题清零

- **资源泄漏**：所有 Cursor 查询均通过 `try-finally` 或 `try-with-resources` 模式确保关闭，彻底消除 Android 应用中常见的 Cursor 泄漏导致的 `android.database.CursorWindowAllocationException`。
- **编译错误风险**：`ResourceParser` 中引用的未定义数组变量已修正，消除编译期符号解析错误。

### 3.2 Critical 级问题清零

- **复杂度降低**：`BackupUtils.exportToText()` 由单一方法（嵌套深度 4+）拆分为 3 个职责单一的方法，认知复杂度降至 10 以下。
- **异常处理规范**：移除对 `NullPointerException` 的捕获，改为对 `File` 对象的前置空检查，符合 Java 异常处理最佳实践。

### 3.3 Major 级问题清零

- **工具类封闭**：`GTaskStringUtils` 和 `ResourceParser` 均设为 `final` 并添加私有构造器，防止无意义实例化。
- **重复代码消除**：`DataUtils` 中两个高度相似的 `exist` 方法合并为统一的 `existsInDatabase()`，降低维护成本。
- **防御性返回**：`getFolderNoteWidget()` 始终返回非空 `HashSet`，调用方可安全执行 `for-each` 遍历而无需前置空检查。
